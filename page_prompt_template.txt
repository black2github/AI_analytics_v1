# ТИПЫ ТРЕБОВАНИЙ И КРИТЕРИИ ПРОВЕРКИ:

## ТИП: 'Функция' (функциональные требования)
- Полнота описания: назначение функции, входные/выходные параметры (типы, форматы)
- Корректность алгоритма: определенность шагов, обработка возможных ошибок, валидация входных данных
- Согласованность: соответствие типам данных, статусным моделям, бизнес-целям
- Техническая осуществимость: в рамках архитектуры, производительность, интеграции
- Качество формулировок: отсутствие двусмысленностей, проверяемость критериев успеха

## ТИП: 'Модель данных' (требования к данным)
- Полнота описания: указание, когда сущность создается и модифицируется
- Полнота метаданных: описание полей, типы данных, ограничения, значения по умолчанию
- Связность: отношения между сущностями, ссылочная целостность
- Качество данных: отсутствие противоречий, дублирования, двусмысленных описаний
- Техническая корректность: поддержка типов данных в СУБД, выполнимость ограничений

## ТИП: 'Процесс обработки' (процессные требования)
- Полнота процесса: нумерация шагов, четкие переходы, обработка всех ветвлений
- Качество событий: именование в форме страдательного причастия, объединение исключающих шлюзов
- Согласованность: логическая целостность процесса

## ДРУГИЕ ТИПЫ (универсальные критерии):
- Полнота, непротиворечивость, корректность, осуществимость, необходимость, недвусмысленность, проверяемость
